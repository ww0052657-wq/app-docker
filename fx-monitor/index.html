This page is served by nginx, which proxies /api and /health to fx-app-spring inside the compose network — so the browser only ever talks to one origin. The trend line holds the last ~90 seconds this page has seen; it starts empty on reload, because the history lives in the browser, not the database.

Feed interval is the gap between the last two batches that actually landed, taken from the server's own captured_at — so it reads about 2s while ACCEPTING is ON. Switch ACCEPTING OFF and nothing is stored at all, so updates stop dead: Last update climbs and goes amber, and the interval simply holds its last value. The orchestrator is still calling every 10s — you can see that in docker compose logs -f fx-orchestrator, not here, because a declined batch leaves no trace in the data. This page polls once a second regardless, and most of those polls find nothing new — which is why the poll count is not the interesting number (0 so far).
